## 1. 기술 스택의 명확성

v0.3에서는 기술 스택을 **구현 가능한 수준으로 구체화**하고, 불필요한 복잡성을 제거하여 단일 개발자/소규모 팀이 실제로 구축할 수 있는 아키텍처로 재설계하였다.

### 1.1 인프라 및 배포 환경

| 항목 | v0.1 (Opus) | v0.3 | 변경 평가 |
|---|---|---|---|
| **인프라 플랜** (CON-3) | Vercel Hobby 플랜 등 완전 무료 티어(Free Tier) | **Vercel Pro + Supabase Pro** (예상 월 $45) | ✅ 현실적. 무료 티어의 Rate Limit/기능 제약이 MVP 운영에 걸림돌이 되는 문제 선제 해결 |
| **월간 인프라 비용** (REQ-NF-019) | 월 0원 (인프라 비용) | **월 $50 이하** | ✅ 실현 가능한 비용 목표로 전환. 무료 티어 내에서의 설계 제약 해소 |
| **캐싱 레이어** | Vercel KV (뱃지 TTL 24h, 환율 TTL 15min) | **Next.js Cache** (뱃지 TTL 24h, `revalidate` 활용) | ✅ 외부 KV 의존성 제거. Next.js 내장 캐시로 단순화하여 추가 비용 절감 |
| **오브젝트 스토리지** | Supabase Storage (라벨 이미지 + OG 이미지) | Supabase Storage (**라벨 이미지만**) | ✅ OG 이미지 동적 생성 삭제로 스토리지 사용량 절감 |
| **백업 전략** (REQ-NF-013) | 일 1회 자동 백업, RPO ≤ 24h | **수동 백업** (Phase 2 자동화) | ⚠️ MVP 현실화이나, 수동 백업 SOP 필요 |

| 항목 | v0.1 (Opus) | v0.3 | 변경 평가 |
|---|---|---|---|

| **환율 API** | 포함 (한국수출입은행, 15분 갱신) | **삭제** | ✅ 환율 동적 계산 로직 및 캐시 관리 부담 완전 제거 |
| **네이버 API** | 컴포넌트 다이어그램에 포함 | **삭제** | ✅ 명시적으로 미사용 API 제거 |

## 2. MVP 목표 및 가치전달 조정 내용

v0.3에서는 **"최소한의 복잡도로 핵심 가치 전달"**이라는 MVP 원칙에 맞게 목표를 현실화하고, 핵심 가치인 '가격 비교'와 '팩트체크'의 전달 경로를 단순화하였다.

### 2.1 프로젝트 스코프 (In-Scope / Out-of-Scope)

| 항목 | v0.1 (Opus) | v0.3 | 변경 의도 |
|---|---|---|---|
| **IS-1 단가 엔진 범위** | iHerb, 쿠팡, 네이버 **3채널** 대상 | **쿠팡 파트너스 단일 채널** (Phase 1) | 핵심 가치(1일 단가 비교)를 단일 채널로 빠르게 검증. 멀티채널은 검증 후 확장 |
| **IS-6 수익 모델** | 제휴 CPA (iHerb, 쿠팡) | 제휴 CPA (**쿠팡 파트너스**) | 수익원을 하나로 집중하여 제휴 파트너 관리 복잡도 감소 |
| **OS-8 (신설)** | 없음 | **다중 채널 병렬 API 오케스트레이션 및 환율 동적 계산 → Phase 2 연기** | 명시적으로 배제 범위를 선언하여 스코프 크리프 방지 |

### 2.2 핵심 기능 요구사항(FR) 조정

| REQ ID | v0.1 (Opus) | v0.3 | 가치전달 영향 |
|---|---|---|---|
| **REQ-FUNC-001** | 3채널 병렬 조회 | **쿠팡 파트너스 단일 조회** | 핵심 가치(자동 단가 비교) 유지. 채널 수는 줄었으나 '수동 계산 대체'라는 가치 핵심은 그대로 |
| **REQ-FUNC-002** | `(가격 × 환율) ÷ 복용횟수` | **`가격 ÷ 복용횟수`** | 동일 가치를 더 단순한 방식으로 전달. 환율 오차 리스크 제거 |
| **REQ-FUNC-003** | 환율 API 연동 (15분 갱신 매매기준율) | **삭제** (KRW 단독) | MVP에서 환율 정확도 보장 부담 제거. 국내 채널 전용으로 가치 충분 |
| **REQ-FUNC-007** | 멀티채널 부분 실패(Partial Failure) 처리 | **삭제** (단일 채널 전환) | 복잡한 에러 핸들링 제거. 단일 채널 장애 시 캐시 폴백으로 대체 |
| **REQ-FUNC-009** | iHerb/쿠팡 딥링크 | **쿠팡 파트너스 딥링크만** | 수익 전환 경로 단일화. A/B 테스트 불필요 |
| **REQ-FUNC-017** | `@vercel/og` 동적 OG 이미지 생성 | **정적 메타태그 + 고정 서비스 로고** | 공유 기능의 핵심(1-Tap 전송) 유지. 시각적 화려함보다 안정성 우선 |
| **REQ-FUNC-021** | URL 복사 + 네이버밴드/문자 대체 공유 버튼 | **URL 클립보드 복사 폴백만** | 폴백 경로 단순화. 대체 채널 SDK 연동 부담 제거 |
